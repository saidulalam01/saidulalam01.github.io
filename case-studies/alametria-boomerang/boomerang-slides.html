The Problem

Churn Was a
One-Way Door

"Gone Forever" Assumption

Every inactive customer was marked churned and forgotten. No recovery model. 130,000 returns were already happening — completely invisibly.

Same Message, Every Segment

A 2-day dropout and a 6-month dropout received identical win-back emails. Timing ignored. Cycle history ignored. Context ignored.

No Early Warning

Drop-offs were only visible after the 31-day mark. By the time we knew, the window to intervene had already closed.

Revenue Left Unmeasured

With no lifecycle tracking, nobody knew that returning customers were already generating 30–35% of total annual revenue.

Primary Risk: Treating all drop-offs as permanent losses — and missing 130,000 recovery opportunities already happening in the data.

The System

Drop + Return +
Escape

State · Drop
The Gap
31+ consecutive days with no purchase or trading activity. Customer enters the recovery window. Not gone — just absent.
Trigger: 31-day silence
State · Return
The Boomerang
Customer re-engages after a Drop. Makes a purchase or resumes trading. Every return opens a new cycle with its own data.
Event: Boomerang ↺
State · Escape
True Churn
Extended absence with no return signal. Tracked separately from active drops — avoids wasted campaign spend on those truly gone.
Signal: No re-engagement
5-cycle tracking: Each customer's Drop → Return journey is tracked across up to 5 complete cycles. Every cycle stores its own LTD bucket, revenue, withdrawals, and purchase count.

The Critical Insight

Coming Back Once Makes
You More Likely to Return

First Return
Cautious Comeback
~30% of all drop-offs return

The baseline boomerang rate. These customers came back on their own — driven by habit, promotion, or platform pull. They need recognition and a loyalty hook to stay for the next cycle.

Win them back. Then keep them.
Third Return+
True Loyalists
~45%+ return rate by cycle 3

Return rates improve with every cycle. A customer who's come back twice is more likely to return a third time than a first-timer is to return at all. They're not flaky — they're loyal in their own rhythm.

Don't treat them like new customers.
The separator is cycle history, not recency. Same gap length. Completely different return likelihood. Sending the same win-back email to both wastes spend on the one who was coming back anyway.
